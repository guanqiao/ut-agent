## 支持企业私有大模型配置

### 问题分析
当前系统对私有 LLM 的支持有限：
- OpenAI/DeepSeek Provider 强制要求 API Key
- Ollama Provider 仅支持 Ollama 协议
- 企业私有模型通常使用 OpenAI 兼容 API，但可能不需要 API Key

### 实现方案

#### 1. 新增 `PrivateLLMProvider` (llm.py)
- 使用 OpenAI 兼容 API（ChatOpenAI）
- API Key 可选（支持无认证或占位符）
- 支持自定义 base_url
- 支持 CA 证书

#### 2. 新增配置项 (config.py)
```python
# 私有 LLM 配置
private_llm_base_url: Optional[str] = None
private_llm_api_key: Optional[str] = None  # 可选
private_llm_model: str = "default"
```

#### 3. 更新文档 (docs/api.md)
- 添加私有 LLM 配置说明
- 添加使用示例

#### 4. 添加测试用例 (tests/test_config.py)
- 测试私有 LLM 配置加载
- 测试无 API Key 场景

### 使用示例
```env
# 企业私有 LLM 配置
DEFAULT_LLM_PROVIDER=private_llm
PRIVATE_LLM_BASE_URL=https://internal-llm.company.com/v1
PRIVATE_LLM_MODEL=company-model-v1
PRIVATE_LLM_API_KEY=optional-key  # 可选
CA_CERT_PATH=/path/to/ca-cert.pem  # 可选
```
